Guía Esencial de Git

Domina el control de versiones desde cero.

1. ¿Qué es Git?

Git es un sistema de control de versiones distribuido. Permite rastrear cambios en los archivos y coordinar el trabajo de varias personas en un mismo proyecto.

2. El Flujo de Trabajo

Git maneja tres estados principales para tus archivos:

  • Directorio de trabajo: Archivos modificados pero no preparados.
  • Área de preparación (Staging): Archivos marcados para incluir en la próxima versión.
  • Repositorio (Commit): Archivos guardados de forma segura en la base de datos local.

3. Comandos Básicos

git init

Crea un nuevo repositorio local en la carpeta actual.

git add .

Agrega todos los cambios al área de preparación (staging).

git commit -m "mensaje"

Crea una instantánea de los cambios con un mensaje descriptivo.

git status

Muestra el estado actual del repositorio (archivos nuevos o modificados).

4. Trabajando con Ramas

Las ramas permiten trabajar en diferentes funcionalidades sin afectar el código principal.

git branch nombre-rama

Crea una nueva rama.

git checkout nombre-rama

Cambia a la rama especificada.

git merge nombre-rama

Fusiona la rama indicada con la rama actual.

5. Repositorios Remotos

git push origin main

Sube tus cambios locales al servidor remoto (como GitHub).

git pull

Descarga e integra los cambios del servidor en tu local.
